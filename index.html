Features

uMod's focus is on modularity and extensibility. The core is highly abstracted and loosely coupled, and could be used to mod any game that uses the .NET Framework or Unity engine, with support for C++ and Unreal powered games on the way.

Open Source

uMod is 100% open source! You can check it out on GitHub and even contribute new features and fixes that you'd like to see included.

Fast Updates

New builds are ready to go for each new game server update usually within an hour or less depending on the game!

Easy to Use

Writing your own plugins is easy! uMod enables you to bring your ideas into games using the easy to learn C# (CSharp) language.

Great Support

Our friendly community staff and members are knowledgeable and readily available to help as needed. Ask away!

FAQ

Got a question? No worries! We've got answers to many of the common and frequently asked questions. If you don't see an answer, don't be afraid to ask!

What is uMod?

uMod is essentially Oxide 3.0 (the previous name); an extended universal API, support for additional games, advanced self-patching, more extensive documentation, a new home, and more to come.

Where can I get plugins?

You can find all of the previous plugins for Oxide at oxidemod.org; these will eventually be updated for uMod along with a standalone site and repository. For new plugins, check out the Plugins section.

When will X game be supported?

A game being supported depends on if it's written in a compatible programming language and isn't jumbled up. Feel free to post a request on our forums or issue tracker and we'll take a look!

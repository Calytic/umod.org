Introduction

The majority of plugins released on umod.org are written in C# (CSharp) which will have filenames ending in .cs. To install the plugin on a server, the server provider must support uMod, else the plugins will not load or do anything.

Getting started

  • 1

    Ensure uMod is installed

    Check that uMod is installed by running the following command in chat or in the server console.
    copy 
    umod.version
  • 2

    Download plugins

    Download freely available plugins from the Plugins page. Do not rename the plugin file or change the file extension when installing a plugin as doing so will make the plugin unusable.

  • 3

    Access to the server's filesystem

    If the server is not hosted locally, connect to the server via an FTP client. If you are unsure of your FTP details, please contact your host.

  • 4

    Ensure the server is started

    Start the server if it is not already running and wait for it to initialize fully. Once started, check that uMod is installed by executing the following chat or console command.
    copy 
    umod.version
    If that command does not work, uMod is not installed. Please return to Step 1.
  • 5

    Navigate to the uMod plugins directory

    After uMod is installed, the default uMod folders (config, data, lang, logs, plugins) are generated. Find the "plugins" folder which is located at umod/plugins by default. Some shared hosting environments may move the plugins folder to a different location.

  • 6

    Install a plugin

    Upload the plugin file into the "plugins" folder and it will be loaded automatically if uMod is currently installed, otherwise please go back to Step 1.

  • 7

    Configure a plugin

    If a plugin is configurable, locate the configuration file by navigating to the "config" directory and edit the JSON file with a name that matches the name of the plugin. If no configuration file is present, then the plugin is either not configurable or it was not successfully loaded and may be broken. In this case, check the latest log files under umod/logs for errors.

  • 7

    Configuration changes

    For the configuration file changes to take effect, simply reload the plugin using the following console command.
    copy 
    umod.reload PluginName
    If the changes are valid and the file was edited correctly, the plugin will now be using the updated configuration.
    If there were errors while reloading the plugin that mention anything related to JSON, then the file was most likely not edited correctly and has invalid JSON.
    When editing configuration files, it's always recommended to validate it using a validator such as www.jsonlint.com.

Data files

Some plugins store persistent data. This data can be found and reset in the umod/data directory. The data files are generally sensibly named with the plugin's name in the filename. Deleting a data file will reset it, but will generally only take effect if the plugin is unloaded or the server is offline. This can be useful when performing a server wipe.

Introduction

Permissions allows server owners to give users unique abilities and benefits on their servers. Administering permissions is easy; simply enter the desired command and you're done! If your server does not have a console, you can use any compatible RCON tool or remote console to send the commands to the server. Most uMod-supported games also support the permission commands in the chat, or will soon.

For this guide, the permission umod.reload will be used, which is used for the umod.reload / reload command that uMod itself provides. Keep in mind that permissions only exist if provided by a plugin or uMod itself.

By default, the groups that are created by uMod are: admin and default. These can be changed by editing those under the umod.config.json file. The admin group will automatically be assigned to players that are recognized as admin by the server. The "default" group will automatically be assigned to ALL players that connect to the server.

Giving an individual player a permissions

Sometimes you just want a single player to have permission to a command or feature.

copy 
umod.grant user Wulf umod.reload

Removing a permission from an individual player

Don't want that user to have the permission you granted anymore? Just revoke it!

copy 
umod.revoke user Wulf umod.reload

Giving an entire uMod group a permission

This is basically the same process only instead of granting a permission to a single user, you grant it to an entire group.

copy 
umod.grant group admin umod.reload

Removing a permission from a group

This is basically the same process only instead of granting a permission to a single user, you grant it to an entire group.

copy 
umod.revoke group admin umod.reload

Adding a user to an existing group

Adding a user to a group will give them all of the permissions assigned to that group

copy 
umod.usergroup add Wulf admin

Removing a user from an existing group

Removing a user from a group will remove from them all of the permissions assigned to that group

copy 
umod.usergroup remove Wulf admin

Adding a entirely new group

copy 
umod.group add vip
copy 
umod.group add vip VIP 0

Removing an existing group

copy 
umod.group remove vip

Setting the title or rank of a group

The group title is usually a short description of a group, sometimes used for chat titles. The rank is a number which sorts a group based on it's importance

copy 
umod.group set vip "[VIP Member]"
copy 
umod.group set vip "[VIP Member]" 1

Setting the parent group of another group

A group will inherit all permissions from it's parent group

copy 
umod.group parent admin default

Showing a user's permissions

copy 
umod.show user Wulf

Showing a group's members and permissions

copy 
umod.show group admin

Showing which user or group has a permission

Sometimes you need a little help tracking down who has a permission, this command should help with that!

copy 
umod.show perm umod.reload

Showing all groups or permissions

To show all of the permission groups, simple use the command below.

copy 
umod.show groups

The show all of the registered permissions from plugins and uMod, use the command below.

copy 
umod.show perms

Using wildcards

A wildcard is something that covers multiple things at once time. For permissions, this is the * symbol. You can use the wildcard (*) to grant multiple permissions at one time. This can be done with all permissions or per plugin based on prefix.

copy 
umod.grant group admin *
copy 
umod.grant user Wulf umod.*

Conclusion

The same commands are also available with the "u." prefix (ex. "u.grant").

That's the basics to permissions for uMod. Permissions give you a fantastic way to manage staff without worrying about them abusing powers from the game's admin functionality (such as flight, noclip, super speed, etc.) so they can still enjoy the game but also help monitor your server at the same time.
